## Permission Code Pattern

All permission codes follow this pattern:
- `{MODULE}_{ACTION}`
- Uppercase letters and underscores only
- Examples: `USER_CREATE`, `TASK_UPDATE`, `CUSTOMER_DELETE`

Common actions:
- `CREATE` - Create new records
- `UPDATE` - Update existing records
- `DELETE` - Delete records
- `VIEW` - View/read records
- `ASSIGN_PERMISSIONS` - Special permission for roles module

## How to Use

### In Frontend Code

```typescript
// Check if user has permission
const hasPermission = user.permissions.some(p => p.code === 'USER_CREATE');

// Check if user has any permission in a module
const hasUserModuleAccess = user.permissions.some(p => p.module === 'users');

// Get all permissions for a module
const userPermissions = user.permissions.filter(p => p.module === 'users');

// Check multiple permissions
const canManageUsers = ['USER_CREATE', 'USER_UPDATE', 'USER_DELETE']
  .every(code => user.permissions.some(p => p.code === code));
```

### In API Endpoints

```typescript
// Use in guards or decorators
@Roles('ADMIN')
@RequirePermission('USER_CREATE')
async createUser() {
  // ...
}
```
